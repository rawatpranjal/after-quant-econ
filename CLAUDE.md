# Structural Economics Library

## Project Overview
Pedagogical library of computational and structural economics models. The active
repo is a tutorial catalog, not a dump of course files. Every active model must
live in a self-contained folder with `run.py` that regenerates `README.md`,
`figures/`, and optional `tables/`.

## Tutorial Contract
Every tutorial folder must have:
- `run.py` as the single executable entrypoint.
- `README.md` generated by `lib.output.ModelReport`.
- `figures/thumb.png` for the root catalog.
- Reproducible outputs from running `python run.py` inside the folder.
- A root `README.md` catalog row with thumbnail, link, method, and key insight.

The README section order is strict:
Overview -> Equations -> Model Setup -> Solution Method -> Results -> Economic
Takeaway -> Reproduce -> References.

## Code Style
- Python 3.11+.
- Prefer JAX (`jax.numpy as jnp`) for numerical kernels when it fits the model.
- NumPy/SciPy are acceptable for estimation, simulation, IO examples, and legacy
  translations where JAX would add clutter rather than clarity.
- Type hints on public helper functions.
- Google-style docstrings or concise equivalent docstrings.
- kebab-case folder names, snake_case file/function names.
- Keep each `run.py` focused on economics, computation, and report generation.

## Shared Library
- `lib/` contains shared utilities: grids, discretization, interpolation, VFI,
  STPFI, simulation, plotting, and output generation.
- Models import from `lib/` via:
  `sys.path.insert(0, str(Path(__file__).resolve().parents[2]))`
- Do not duplicate utility code in model folders when a shared helper already
  exists.
- If a second tutorial needs the same nontrivial helper, promote it to `lib/`.

## Output Standard
Every model `run.py` produces:
- `README.md`: auto-generated via `lib.output.ModelReport`.
- `figures/*.png`: matplotlib, 150 DPI, academic style.
- `tables/*.csv`: optional, if the model has tabular results.
- `figures/thumb.png`: 200x150 thumbnail for the root catalog.

After adding or changing tutorials, run:

```bash
python scripts/validate_catalog.py
```

## Legacy Policy
- Active topic folders should contain polished tutorial folders only.
- Original notebooks, PDFs, MATLAB/R scripts, and course handouts belong under
  `_legacy/`.
- Do not delete source material. Move it into `_legacy/` when cleaning active
  folders.
- New work should not depend on files in `_legacy/` at runtime.

## Do Not
- Do not create new Jupyter notebooks in active folders.
- Do not leave `.ipynb_checkpoints` outside `_legacy/`.
- Do not add interactive or Plotly visualizations. Use matplotlib only.
- Do not add dependencies beyond `requirements.txt` / `pyproject.toml` without
  discussion.
- Do not add a tutorial that cannot regenerate its README, figures, and tables
  from `python run.py`.
